解决浏览器无法正确复制带WebKitBoundary表单数据请求为curl命令的问题

# 背景

在写爬虫时，往往会需要将浏览器开发者工作中的网络请求复制成 curl 命令，然后结合 vscode 的 curlconverter 插件，可以很方便快捷的将请求转为相应语言的代码，但是到目前为止 Firefox 和 Chrome （ Edge 未测试）对于带 WebKitBoundary 表单数据的请求都不能被浏览器正确识别并复制。
这是 curlconverter 官方仓库中网友提的 ISSUE

# 解决办法

  1. 从浏览器复制带 WebKitBoundary 表单数据请求为 curl 命令；
  2. 将该 curl 命令导入到 postman ，导入时选择 RAW text 数据导入；
  3. 在 postman 中测试该请求是否能够正常响应；
  4. 使用 postman 生成该请求的对应编程语言代码。
